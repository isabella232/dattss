How it Works?

Since we're using HTTP (instead of UDP for statsd and other simialr solutions), the client library pre-aggretages the statistics it receives before sending it periodically every 5s to a DaTtSs server. Each process running a DaTtSs client library only make one HTTP request every 5s which is super lightweight and won't overcharge your systems. DaTtSs is therefore able to aggregate an unlimited number of statistics at very high frequencies.
